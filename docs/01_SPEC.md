# 제품 스펙

## 서비스 요약

이 서비스는 개인용 홈쇼핑 판매 일정 및 출고 일정 관리 도구입니다.

사용자는 상품 공급 업체, 홈쇼핑 채널, 관련 파트너 사이에서 전달과
정리를 맡는 중간 운영 역할을 합니다. 현재는 특정 제품이 언제 어느
홈쇼핑에서 판매되었고, 해당 제품이 언제 출고되는지를 다이어리에 수기로
입력해 관리하고 있습니다. 이 방식이 점점 버거워지고 있기 때문에, 웹
입력기를 통해 데이터를 구조화해서 저장하고 Google Sheets와 Google
Calendar로 공유하는 것이 목표입니다.

이 서비스는 보통 2-3명 정도의 허용된 사용자만 사용하는 매우 개인적인
업무 도구입니다.

## 주요 사용자

- 상품 판매 및 출고 일정을 등록하는 내부 운영자.
- 공유된 Google Sheets 또는 Calendar를 확인하는 관련자.
- 추후 제한적으로 앱 접근 권한을 받을 수 있는 협업자.

## 핵심 업무 흐름

1. 운영자가 공급 업체나 파트너에게서 일정 정보를 전달받습니다.
2. 운영자가 웹 앱에 접속해 Google 계정으로 로그인합니다.
3. 앱은 해당 Google 이메일이 허용된 사용자인지 확인합니다.
4. 운영자가 제품, 홈쇼핑 채널, 판매일, 출고일 정보를 입력합니다.
5. 앱은 입력된 정보를 자체 데이터베이스에 저장합니다.
6. 앱은 관련 정보를 Google Sheets와 Google Calendar에 작성합니다.
7. 관련자는 공유된 Sheets와 Calendar에서 일정을 확인합니다.

## MVP 범위

MVP는 "가장 작지만 실제로 쓸 수 있는 첫 버전"을 의미합니다. 이
프로젝트에서는 복잡한 자동화보다 현재의 수기 다이어리 업무를 먼저
대체하는 데 집중합니다.

첫 MVP에 포함할 기능:

- Google 로그인과 허용 이메일 기반 접근 제어.
- 최근 일정, 예정 출고, 동기화 상태를 보여주는 대시보드.
- 판매 및 출고 일정 등록 폼.
- 일정 목록과 검색.
- 기본적인 홈쇼핑 채널 및 업체 관리.
- Cloudflare D1을 이용한 자체 데이터 저장.
- 일정 등록 성공 후 Google Sheets 행 생성 및 Google Calendar 일정 생성.
- Google 작성 성공/실패를 확인할 수 있는 동기화 로그 또는 상태 패널.

MVP에서 제외할 기능:

- Google과의 완전한 양방향 동기화.
- 복잡한 권한/역할 관리.
- 고급 충돌 해결.
- 대량 가져오기/내보내기.
- 자동 알림 워크플로우.
- 모바일 우선 재설계.

## 디자인 방향

현재 Stitch 초안의 분위기를 계승합니다.

- 따뜻한 톤의 전문적인 대시보드 스타일.
- 좌측 고정 사이드바.
- 밀도는 높지만 읽기 쉬운 테이블 화면.
- 코랄 계열의 주요 액션 색상.
- 따뜻한 오프화이트 배경 위의 흰색 카드.
- 마케팅 랜딩 페이지가 아닌 데이터 중심 업무 화면.

업로드된 Stitch 화면은 고정된 픽셀 명세가 아니라 시각적 초안입니다. 실제
구현에서는 이 분위기를 유지하되, 실제 데이터 구조와 반응형 동작에 맞게
레이아웃, 간격, 컴포넌트를 조정합니다.

## 추천 기술 스택

- 프론트엔드: React 19 + Vite + TypeScript.
- 스타일: Tailwind CSS.
- 아이콘: lucide-react.
- 폼: React Hook Form + Zod.
- 호스팅: Cloudflare Pages.
- API 계층: Cloudflare Pages Functions 또는 Workers.
- 데이터베이스: Cloudflare D1.
- 인증: Cloudflare Functions에서 Google OAuth 직접 구현.
- 접근 제어: Cloudflare Secret 또는 환경 변수에 저장한 허용 이메일 목록.
- Google 연동: Google Sheets API, Google Calendar API.

## 인증 동작

앱은 인증 전 실제 업무 데이터를 노출하지 않습니다.

인증되지 않은 사용자는 최소한의 Google 로그인 화면만 볼 수 있습니다.
Google OAuth가 완료되면 백엔드에서 이메일 허용 여부를 검사합니다. 허용된
이메일이 아니면 앱 접근을 거부합니다.

Google 로그인 흐름에서는 Sheets와 Calendar API 권한도 함께 요청해서,
사용자가 앱 로그인과 Google 연동을 하나의 행위로 느끼도록 설계합니다.

## 운영 제약

- Cloudflare 무료 또는 저비용 운영을 목표로 합니다.
- 인증되지 않은 요청은 D1 조회나 Google API 호출을 발생시키지 않습니다.
- 정적 자산은 Cloudflare Pages가 직접 제공해야 합니다.
- 서버 함수는 `/api/*` 경로에서만 실행되도록 설계합니다.
- OAuth callback은 Google token 교환 전에 반드시 `state`를 검증합니다.
- Google refresh token과 앱 secret은 절대 브라우저에 노출하지 않습니다.
